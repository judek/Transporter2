    For anyone who needs it, on the inline worker full example
    var bb = new BlobBuilder();
    bb.append(document.querySelector('#worker1').textContent);

    // Note: window.webkitURL.createObjectURL() in Chrome 10+.
    var worker = new Worker(window.URL.createObjectURL(bb.getBlob()));
    should now be
    var bb = new Blob(document.querySelector('#worker1').textContent.split());

    // Note: window.webkitURL.createObjectURL() in Chrome 10+.
    var worker = new Worker(window.URL.createObjectURL(bb));

  • Stephan Beal

    Under "use cases": "Code syntax highlighting or other real-time text formatting"

    If the worker cannot touch the DOM (as is stated above) then that use case would seem to have little use. To get around it we would have to copy all text we want to highlight into the worker and wait for it to send us back the whole text. Doing incremental syntax highlighting this way would not (IMO) be practical.

    Similar applies to the image filtering: if a worker cannot touch the canvas then it can't do much filtering on it unless perhaps we wedge a layer between the canvas and the worker.
